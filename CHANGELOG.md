# 桌面机器人 - 版本记录

## v2.6 - 2026-03-26

### 修复
- **Sleepy 唤醒机制** (`main.py`)
  - 语音对话完成后 sleepy → idle（之前只重置计时器，不改表情）
  - 新增 webchat 对话检测：webchat 回复时 sleepy → idle
  - 通过 `webchat_active.txt` 文件同步 webchat 会话状态
  - 其他触发（超声波、提醒、早报）不受影响

## v2.5 - 2026-03-25

### 新增
- **歌词显示** (`ncm_api.js`, `music_player.py`)
  - `ncm_api.js` 新增 `lyric` 命令，调网易云 /song/lyric 接口
  - 歌词解析(_parse_lrc)、异步获取(_fetch_lyrics)、播放位置同步
  - 播放器 widget 扩大 h=120→240，歌词居中当前行，长歌词自动换行
  - 进度条常驻在歌词下方，无时间数字
  - 无歌词显示"暂无歌词"，加载中显示"歌曲加载中"
- **摇杆音乐控制** (`main.py`, `music_player.py`)
  - PS2 摇杆从表情控制改为音乐控制
  - 按下 → 播放/暂停 + happy 表情
  - 左推回中 → 上一首 + angry 表情
  - 右推回中 → 下一首 + love 表情
  - 顺时针转 → 快进10s + surprised 表情
  - 逆时针转 → 快退10s + shy 表情
  - 转圈检测：600ms 窗口内 3+ 次连续方向转换
  - `music_player.py` 新增 fast_forward/fast_rewind 方法
  - `_seeking` 标记防止 seek 时误触发自动下一首

### 优化
- **播放器 widget 布局**
  - 歌单名/歌名/歌手间距拉大，不再挤在一起
  - 进度条 6→8px，未播放部分高对比灰色 (90,90,120)
  - 进度条和按钮之间留 10px 间距不重叠
  - 4 个按钮居中对称排列，间距 60px
- **摇杆防误触**
  - 左右推需回中心才触发（推到底→松手→触发）
  - 最少推出 80ms 防抖
  - 转圈检测到后取消待触发的单次操作

## v2.4 - 2026-03-24

### 新增
- **情绪感知** (`chat_bridge.py`)
  - 关键词检测用户情绪：疲惫/开心/难过/烦躁/焦虑
  - 检测到情绪后注入 OpenClaw prompt，自动调整回复风格
  - 疲惫→简短温暖；难过→安静陪伴；烦躁→克制安抚
- **偏好学习** (`chat_bridge.py`)
  - 自动从对话中学习用户偏好，存入 `memory/preferences.json`
  - 支持：温度偏好、音乐风格、作息时间
  - 说出"喜欢XX度/XX音乐/X点睡觉"即自动记录
- **天气突变提醒** (cron)
  - 每天 8:00/12:00/17:00 检查深圳天气
  - 下雨/骤降5°C+/大风时主动提醒带伞/加衣
- **GitHub 监控** (cron)
  - 每天 10:00/14:00/19:00/22:00 检查 desktop-robot 仓库
  - 有新远程提交或本地未推送时提醒

## v2.3 - 2026-03-24

### 新增
- **实时功率显示** (`status.py`)
  - 读取 RPi 5 PMIC 各电源轨电压×电流，计算总功率
  - 显示在 SYSTEM 面板底部，3秒缓存防闪烁
  - 颜色编码：<8W 绿色 / 8-15W 黄色 / >15W 红色
  - 无需额外硬件，`vcgencmd pmic_read_adc` 直接读取

### 优化
- **语音识别大幅升级** (`voice_pipeline.py`)
  - Whisper 模型从 `tiny`(39M) 升级到 `small`(244M)，中文准确率显著提升
  - 添加 `initial_prompt` 引导简体中文输出，减少繁体混入
  - 添加 `condition_on_previous_text=False` 防止上下句干扰
- **音乐播放器 UI 改版** (`music_player.py`)
  - 歌名字体 15→24px 粗体（与左侧日期同级），歌手 12→16px 粗体
  - 使用 `get_font` 统一字体渲染，与左侧面板风格一致
  - 播放器边框更亮 (90,90,120)→(130,130,170)
  - 按钮半径 18→20，底色/边框提亮
  - 随机/顺序按钮下方字母删除，widget 高度 100→120px
- **SYSTEM 面板布局** (`status.py`)
  - 新增"实时功率"行，行间距调整为 28px
  - 标题间距 28→24px，避免与底部按钮重叠

### 修复
- 提醒正则支持"的"连接词："帮我记一个两分钟后的提醒"之前匹配失败
- 提醒正则支持中文数字："两分钟"之前因 `\d+` 不匹配"两"而失败
- 提醒消息末尾标点自动清理（句号、逗号等）
- `config.txt` 添加 `audio_pwm_mode = 2` 减少音频干扰

## v2.2 - 2026-03-24

### 新增
- **定时提醒系统** (`reminder.py`)
  - 语音设置：支持"提醒我X分钟后Y"、"X点提醒我Y"、"提醒我下午X点Y"等多种口语
  - 自动繁体转简体（Whisper 可能输出繁体）
  - 提醒卡片：覆盖在脸区域，显示内容 + 知道了/再等5分钟 按钮
  - 叮咚提示音 ×3（numpy 合成，1.6秒）
  - 60秒无操作自动关闭，支持推迟5分钟
  - 持久化存储 `reminders.json`
- **早报自动播报** (`briefing.py`)
  - PIR 检测到人 + 6:00-11:00 + 今天未播报 → 自动触发
  - 播报内容：问候 + 深圳天气（中文，TTS 友好格式）
  - 天气数据来自 wttr.in，风向符号转中文、温度单位转"度"
  - 播报状态持久化 `briefing_state.json`（每天只播报一次）
- **PIR 人体感应逻辑调整**
  - 移除 sleepy→happy 唤醒表情
  - sleepy 表情仅与聊天活动（1小时无对话）关联
  - 屏幕开关仅与 PIR 关联，不再受聊天活动影响

### 修复
- 天气获取因代理 (Mihomo 7890) 返回 502，在 `status.py` 和 `briefing.py` 的 curl 命令中添加 `--noproxy "*"`

## v2.0 - 2026-03-23

### 新增
- **电位器音量控制**：MCP3008 ADC + 电位器，实时调音量
  - 滑动平均滤波，变化 ≥3% 才触发
  - 端点补偿，确保 0%~100% 覆盖
  - 底部水平音量条显示（绿色填充 + 百分比）
- **PS2 摇杆表情控制**：上下左右切表情
  - ⬆️ happy / ⬇️ sad / ⬅️ angry / ➡️ love / 🎯 surprised
  - 按键走 GPIO12 内部上拉，方向走 MCP3008 CH1/CH2
- **SPI 启用**：`dtparam=spi=on`，使用 `spidev0.0` (CE0)
- 删除屏幕音量 +/- 按钮，退出按钮移至右下角

### 修改
- 移除屏幕音量按钮，音量完全由电位器控制
- 音量条从竖向改为水平，位置在摄像头按钮右侧
- 退出按钮从中间移至最右下角

- 语音对话：按住按钮/按键录音 → Whisper STT → OpenClaw → TTS
- 网易云音乐集成 + 屏幕播放器 Widget
- 语音-音乐联动（录音自动暂停/续播）
- HA 智能家居集成
- 桌面启动器
